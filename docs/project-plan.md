# 项目计划

## 1. 项目目标

建设一个可用于赛事初筛的学生自助抽签系统，完成学生信息登记、个人抽奖、35 人动态筛选、实时统计展示，并提供可演示的抽卡动画体验。

## 2. 范围定义

### 本期范围

- 学生信息登记：姓名、学号、头像
- 批量导入名单
- 候选池人数达到数百人时可正常使用
- 35 位选手筛选
- 每位学生默认 10 次个人抽奖机会
- 动态概率保底 35 个名额
- 实时剩余人数统计
- 抽签历史与结果展示
- Demo 级视觉动画
- 自动化测试与本地可运行验证

### 暂不纳入

- 多角色权限管理
- 后端数据库
- 多轮赛事编排
- 审计日志与正式上线运维体系

## 3. 里程碑

1. 需求梳理与产品文档
2. 前端 Demo 开发
3. 抽签规则与交互测试
4. 构建验证并演示

## 4. 交付物

- 项目计划文档
- 产品需求文档
- 可运行前端 Demo
- 自动化测试脚本
- 构建与测试结果

## 5. 风险与应对

- 风险：几百人数据下动画和渲染卡顿
  应对：仅对关键区域重绘，抽签逻辑与视图分层
- 风险：用户误操作导致抽签结果丢失
  应对：启用本地持久化和“重置确认”机制
- 风险：概率规则理解偏差
  应对：在界面中明示“当前中奖率 = 剩余名额 / 剩余总抽奖次数”

## 6. 验收标准

- 能登记、导入并管理数百位学生
- 能从候选池中抽出 35 位不重复学生
- 抽取后能实时显示剩余待抽人数和剩余未中签人数
- 抽签结果刷新页面后仍能保留
- 自动化测试通过
- 本地构建成功
