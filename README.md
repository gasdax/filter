# 星辉参赛祈愿

一个面向校园报名筛选场景的多人在线抽奖系统。学生自行注册并登录，管理员通过后台维护可注册学号白名单、开放抽奖、查看注册信息和调整抽奖次数，最终动态筛选出 35 位中签者。

系统包含全屏过场动画、学生独立登录页、管理员独立后台，以及适合活动现场展示的抽奖结果演出效果。

## 当前功能

- 学生自行注册：填写学号、密码、姓名、手机号码，并可上传自定义头像
- 学号白名单校验：只有管理员导入的学号才允许注册
- 学生独立登录：学生登录成功后进入个人抽奖主页
- 管理员后台：通过 `/admin` 进入管理员登录页和管理面板
- 抽奖开关：学生注册成功后不能立刻抽奖，必须由管理员手动开放
- 抽奖次数控制：每位学生默认 20 次机会，管理员可单独调整
- 动态概率：注册人数不超过 35 人时理论中奖率为 100%，超过 35 人后按剩余名额和剩余次数动态计算
- 全屏过场动画：注册成功/失败、抽奖结果均可播放全屏视频过场
- 并发优化：服务端使用内存态数据和串行写操作，较适合校内活动级别并发

## 页面入口

- 学生首页：`http://127.0.0.1:4173/`
- 管理员页面：`http://127.0.0.1:4173/admin`
- 后端健康检查：`http://127.0.0.1:4174/api/health`

## 默认账号

## 管理员使用流程

1. 打开 `http://127.0.0.1:4173/admin`
2. 使用管理员密码登录
3. 导入或粘贴允许注册的学号白名单
4. 等待学生自行注册
5. 视活动安排手动点击“开放抽奖”
6. 在后台查看已注册学生、调整个人抽奖次数、重置中签结果或清空注册数据

## 学生使用流程

1. 打开学生首页
2. 填写学号、密码、姓名、手机号码并完成注册
3. 注册结果会先播放过场动画，再展示成功或失败页面
4. 注册成功后可前往学生登录页登录
5. 管理员开放抽奖后，学生才能在个人页面进行抽奖

## 数据与实现说明

- 数据文件为根目录下的 `server-db.json`
- 服务端启动时会把数据读入内存，并进行异步落盘
- 所有写操作都会串行执行，减少多人同时操作时的状态竞争
- 头像当前以前端 Base64 形式提交，建议控制在 2.5MB 以内

## 主要文件

- `src/app.ts`：前端页面状态、交互逻辑、路由切换
- `src/style.css`：页面样式和过场动画展示样式
- `server.mjs`：注册、登录、抽奖、管理员接口与服务端状态管理
- `src/utils/lottery.ts`：中奖概率与抽奖规则
- `src/app.test.ts`：前端关键流程测试
- `src/utils/lottery.test.ts`：抽奖规则测试

## 注意事项

- `server-db.json` 是本地运行数据，不建议直接作为正式生产数据库方案
- 当前版本适合演示、校内活动和中小规模并发场景
- 如果后续要进一步提升稳定性，建议把存储从 JSON 文件迁移到 SQLite 或其他数据库
